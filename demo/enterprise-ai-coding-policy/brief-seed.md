# Enterprise AI Coding Policy Brief

Status: draft for leadership review
Audience: engineering leadership, security, legal, and customer-facing teams

## Recommendation

Adopt an approved AI coding assistant policy for engineering, starting with a
small pilot and clear review requirements.

The policy should let engineers use AI tools for routine coding help while
keeping customer data, secrets, proprietary strategy, and production changes
under human control.

## What customers are asking for

Enterprise customers increasingly ask whether AI tools touch their data, code,
or support artifacts.

The current answer is inconsistent because usage is handled by individual
teams instead of a single company policy.

## Risks and constraints

The main risk is accidental disclosure of sensitive information through prompts
or generated code that has not been reviewed.

The policy needs to be simple enough that engineers actually follow it.

## Proposed rollout

Start with a small engineering pilot using approved tools only.

Require human review before AI-assisted code reaches production.

## Open questions

- Which vendors are approved for source code use?
- What logging or audit evidence do enterprise customers expect?
- How should exceptions be requested?
